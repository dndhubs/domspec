# Security Policy

## Supported Versions

Unless otherwise stated in the package README:

| Version line | Status |
| --- | --- |
| latest major | supported |
| previous major | security fixes only |
| older versions | unsupported |

## Reporting a Vulnerability

Please **do not** report security vulnerabilities through public GitHub issues.

Instead, report them privately to:

- **jetroolowole@gmail.com**

If private vulnerability reporting is enabled in GitHub, you may also use the repository Security tab.

## What to Include

Please include as much of the following as possible:

- affected package name and version
- affected environment and runtime
- vulnerability type
- reproduction steps
- proof of concept if available
- impact assessment
- relevant files, commits, tags, or URLs
- any mitigations or workarounds already identified

## Response Targets

We aim to:

- acknowledge receipt within **2 business days**
- provide an initial triage update within **5 business days**
- coordinate remediation and disclosure responsibly

## Disclosure

We prefer coordinated disclosure. Please give us a reasonable opportunity to investigate and remediate the issue before public disclosure.

## Scope Notes

For public OSS packages, this policy applies to code published in the repository and package registry.

For gated or internal packages, vulnerability handling may include additional internal procedures.
